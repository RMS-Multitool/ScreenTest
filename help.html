1. Overview

ScreenTest is a browser-based LED screen preview tool built for FOHP Australia. It lets you design and test content layouts for the venue's 9m × 3m LED screen (3456 × 1152 px, 3:1 aspect ratio) before show day — so you know exactly how your content will look before it goes live.

All uploaded files are stored locally in your browser only — nothing is sent to any server. Clearing your browser cache will permanently delete all uploaded media and custom presets. Use Export Template and Export PIPs regularly to back up your work.

  • Design and preview background and foreground (PIP) content layouts
  • Upload images and videos to see how they fill the screen
  • Create and save custom PIP presets with precise positioning
  • Export professional layout documents to share with clients or crew
  • Works best in Chrome or Edge

2. The Screen Preview

The large area on the right side of the app is the live preview. It updates in real time as you change settings.

  • Corner registration marks — The blue crosshairs in each corner are alignment guides. They are not part of the content and won't appear on the actual screen.
  • Metadata bar — The bar below the preview shows: current resolution (e.g. 3456 × 1152), physical screen size (9m × 3m), the preview zoom scale, and the name of the active PIP preset.
  • Fullscreen mode — Click the ⛶ Fullscreen button in the top-right corner for an immersive full-window preview. Press Escape or click anywhere to exit.
  • Safe Area toggle — Enable this in the Screen panel to show a dashed yellow border 5% in from each edge. Content inside this border is safe from edge distortion on the physical screen.
  • PIP Border toggle — Shows a green outline around the active PIP zone so you can see its exact boundary in the preview.

3. Background Layer

The background fills the entire 9m × 3m screen behind any foreground overlay.

Uploading media

To upload a background file, click the upload area or drag a file directly onto it. Supported formats: JPGPNGGIFMP4WebM. Maximum file size: 500 MB. WebP is not supported — convert to PNG or JPG first.

Files are stored locally in your browser's storage (IndexedDB). They persist across page refreshes and browser restarts, but will be lost if you clear your browser cache or site data.

Object Fit — controls how the media fills the screen

  • Cover (default) — Scales and crops the media to fill the screen completely, preserving aspect ratio. Some edges may be cropped.
  • Contain — Fits the entire media within the screen. Black bars may appear on the sides or top/bottom.
  • Fill — Stretches the media to exactly fill the screen regardless of aspect ratio.

Holding Images

The Holding Images section contains pre-loaded test patterns and templates. Click any to preview it instantly — great for checking alignment before you have final content. An active holding image has a green border. Click it again to deselect.

Clear

Removes the current background without deleting the file from the library.

4. PIP / Overlay Layout

This panel controls the layout of the foreground overlay — where on screen the PIP zone appears.

Preset cards — Click any card to apply that layout instantly. The preview updates immediately with a smooth transition.

Built-in presets

  • Background Only — No foreground overlay is shown. The full screen displays background content only.
  • Foreground Full — The PIP zone fills the entire screen. Useful when your foreground content should fill everything.

Custom presets created by your admin also appear here.

Admin mode

A lock icon (🔒) appears in the Custom Presets section header. Click it and enter the admin code to unlock preset creation and management. Contact your system administrator for the code.

Do not share the admin code. Admin mode allows creating, editing, and deleting presets.

5. Foreground / PIP Content

The foreground layer is the media that plays inside the PIP zone defined by your active layout preset.

Uploading works the same as the background layer — click or drag to upload. Supported formats: JPGPNGGIFMP4WebM. WebP is not supported.

Object Fit defaults to Contain for foreground content, so your media fits neatly inside the PIP zone without cropping.

If you load a foreground file while the layout is set to 'Background Only', the app will automatically switch to 'Foreground Full' so your content becomes visible straight away.

Clear

Removes the current foreground without deleting the file from the library.

6. Brand Assets

Brand Assets are a dedicated library for logos, sponsor images, and branded graphics you use regularly across different events.

Supported formats: JPGPNGSVG (no video, no WebP).

Managing assets

Hover over any asset card to reveal three action buttons:

  • Set as BG — Use the asset as the background layer
  • Set as FG — Use the asset as the foreground / PIP content
  • Remove — Permanently delete the asset from storage

Brand assets also appear in the main Media Library, so you can access them from either panel.

7. Media Library

The Media Library shows every image and video you have uploaded (brand assets appear in both the Brand Assets panel and here).

Each card shows:

  • A thumbnail preview
  • A type badge: IMG for images, ▶ VID for videos, plus the file size
  • On hover: BG, FG, and ✕ (delete) buttons

Active items are highlighted — a blue border means it is currently used as the background, a green border means it is the active foreground.

Items are listed newest first. Deleted items are permanently removed from browser storage.

8. The PIP Builder

The PIP Builder lets you create custom PIP presets with precise, pixel-accurate positioning. It requires admin mode to be unlocked.

To open: With admin mode unlocked, click the + New PIP button.

The canvas

The canvas represents the full 9m × 3m venue screen. The green box is your PIP zone. Drag the centre of the box to move it. Drag any of the 8 handles (corners and edges) to resize it.

Guide overlays on the canvas:

  • Rule of thirds lines — Divide the screen into 9 equal sections
  • Centre crosshair — Marks the exact centre of the screen

Dual PIP mode

Tick "Add second PIP slot" to add a second zone (shown in orange). Both slots are independently positioned and resized. Click a slot to make it active — the info panel below updates to show that slot's dimensions.

Info panel

Below the canvas, four values update in real time showing the active slot's:

  • Width and Height — as percentage of screen and in physical metres
  • From Left and From Top — offset position as percentage and metres

Example: Width 50% = 4.50m means the PIP zone is half the screen width, which is 4.5 metres on the physical screen.

Aspect ratio lock

Enter your content's pixel resolution (e.g. 1920 width, 1080 height) and tick Lock ratio. Resizing will now maintain that aspect ratio automatically. The simplified ratio (e.g. 16:9) is displayed next to the inputs.

Keyboard nudge

For fine positioning, click on the canvas area (not inside a text input), then use:

↑ ↓ ← → Move 0.5% per press
Shift + ↑ ↓ ← → Move 0.1% per press

Arrow key nudging doesn't work if a text input field is focused. Click on the canvas area first, then use the arrow keys.

Saving

Enter a preset name (up to 40 characters) and click Save Preset. The preset appears in the PIP Layout panel immediately and is stored in your browser.

9. Exporting & Screenshots

Export Template

Generates a professional PNG layout document showing the full screen configuration. Click Export Template in the top bar.

The exported file includes:

  • Screen dimensions (3456 × 1152 px) and physical size (9m × 3m)
  • A pixel grid for reference
  • All PIP zones with their exact sizes (pixels, percentage, metres) and positions
  • FOHP-branded header and footer
  • The active preset name

The file downloads as FOHP-Template-[PresetName].png. Use this to share layout plans with clients, production crew, or for printed documentation.

Screenshot

Click 📷 Screenshot to capture the current preview at full 3456 × 1152 resolution with all layers composited exactly as shown. Downloads as a PNG. Use this to show clients how their content will look on the physical screen.

Screenshots reflect exactly what's shown in the preview — including Object Fit settings. Make sure your layers are configured correctly before exporting.

10. Tips & Troubleshooting

My uploaded files have disappeared.

All media is stored locally in your browser. Clearing browser cache or site data permanently removes all uploaded files. Before clearing your cache, download any important files directly. Back up custom presets using the Export PIPs option in admin mode.

WebP files won't upload.

WebP is not supported. Convert your image to PNG or JPG using any image editor (such as Preview on Mac, Paint on Windows) or a free online converter, then upload again.

Arrow keys aren't nudging the PIP box.

Click somewhere on the builder canvas (not inside a text field like the name or resolution inputs), then try the arrow keys again.

The fullscreen view looks different from the preview.

The fullscreen view scales the preview to fit your monitor. Most monitors are 16:9 — much taller relative to width than the actual 3:1 venue screen. The proportions shown in the regular preview area are more representative. Use fullscreen to check how media fills the space, not to judge proportions.

Custom presets have disappeared.

Custom presets are stored in your browser. They are lost if you clear site data. Back them up regularly using Export PIPs in admin mode. Restore them with Import PIPs.

Video isn't playing in the preview.

Videos are muted by default due to browser autoplay rules — this is normal. If a video doesn't play at all, try a different browser (Chrome or Edge recommended). Very large video files may be slow to load in the preview; this does not affect actual playback on the venue system.

The preview scale is very small.

The preview scales to fit the available window space. The screen is 3:1 — much wider than tall — so it will always appear narrow on a standard monitor. Use the fullscreen button for the largest possible view.
